## 큰 수 만들기

## 전략
### 전략 1: 단조 감소 스택 (원본 문자열 2회 순회)
- 인덱스에 해당하는 값을 기준으로 단조 감소하는 스택을 만들고, 제외할 인덱스 목록을 만들기
- 단조 감소 스택에 인덱스를 담고, 스택에서 pop하는 경우 해당 인덱스를 제외할 인덱스 목록에 추가
- 제외할 인덱스 목록을 정렬
- 결과를 담을 빈 문자열을 만듦
- 원본 문자열을 순회하며 제외할 인덱스 목록에 포함된 인덱스의 경우 결과 문자열에 추가하지 않음
- 순회 중 결과 문자열의 길이가 (원본 문자열 길이 - k)와 같아지면 결과 문자열을 반환

### 전략 2: 단조 감소 스택 (원본 문자열 1회 순회) (🗒️)
- 전략 1처럼 단조 감소 스택을 만드는데, 인덱스 대신 값을 담음
- 그리고 pop 조건에 지금까지 pop한 횟수가 k보다 작아야 하는 조건을 추가
- 순회가 끝났을 때, pop한 횟수가 k보다 작은 경우를 처리

## 시도했으나 실패했던 풀이
- 뒤의 숫자보다 작은 숫자가 보이면 그 숫자를 제거하여 문자열을 다시 순회하는 방법
    - 시간 복잡도 ${O(N^2)}$의 풀이
    - 같은 숫자 100만 개로 구성된 숫자와, k가 99만 9999인 테스트 케이스에서 시간 초과 발생
